  <action>
1. Add a `make e2e` target to the Makefile that:
   - Runs `cd frontend && npx playwright test --reporter=list`
   - Add it to the .PHONY list at the top

2. Add an `e2e` job to `.github/workflows/ci.yml` that:
   - Name: "E2E -- Playwright"
   - runs-on: ubuntu-latest
   - needs: [backend-test, frontend-build] (run after unit tests and build pass)
   - Steps:
     a. actions/checkout@v4
     b. Setup Python 3.12 with uv (same pattern as backend-test job)
     c. Install backend deps: `cd backend && uv sync --all-extras`
     d. Setup Node.js 22 with npm cache (same pattern as frontend-test job)
     e. Install frontend deps: `cd frontend && npm ci`
     f. Install Playwright browsers: `cd frontend && npx playwright install --with-deps chromium`
     g. Create .env file with test defaults:
        ```
        echo "SECRET_KEY=$(openssl rand -hex 32)" > .env
        echo "DEFAULT_USERNAME=admin" >> .env
        echo "DEFAULT_PASSWORD=changeme" >> .env
        echo "DATABASE_URL=sqlite+aiosqlite:///./test_e2e.db" >> .env
        ```
     h. Run Alembic migrations: `cd backend && uv run alembic upgrade head`
     i. Start backend in background: `cd backend && uv run uvicorn app.main:app --host 0.0.0.0 --port 8000 &`
     j. Build and start frontend in background:
        ```
        cd frontend && npm run build && npm run start &
        ```
     k. Wait for servers: use a loop that curls localhost:8000/api/v1/health and localhost:3000 with retries
     l. Run Playwright: `cd frontend && npx playwright test --reporter=list`
     m. Upload Playwright report as artifact on failure using actions/upload-artifact@v4

Do NOT modify any existing jobs. Only append the new e2e job.
  </action>
